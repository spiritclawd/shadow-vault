The Problem

Every trade reveals your edge

On-chain transparency is a feature for verification — and a vulnerability for strategy. When AI agents trade in public, their logic is extracted before their transactions land.

🔓

Strategy Extraction

Any observer can reverse-engineer agent logic from public transaction patterns, draining alpha within hours of deployment.

👁️

Front-running

Searchers monitor mempool to copy or sandwich agent trades, capturing value that should belong to the strategy owner.

🚫

Censorship Risk

Validators can selectively exclude transactions based on identifiable patterns, rendering autonomous agents unreliable.

How it works

Trade without exposure

Shadow Vault uses Pedersen commitments and zero-knowledge proofs to hide trade parameters until execution is irreversible.

01

Commit

Agent hashes trade parameters into a Pedersen commitment on-chain. The strategy remains hidden.

02

Prove

A ZK-proof verifies the commitment satisfies vault constraints without revealing the underlying values.

03

Execute

The trade executes atomically via Jupiter. Only then are parameters revealed — after extraction is impossible.

04

Settle

Assets return to the encrypted vault. Proof verification completes. The strategy remains yours alone.

const vault = await ShadowVault.init({
  connection,
  wallet,
  programId: 'ShdwV...'
});

// Commit without revealing
const commit = await vault.commit({
  mintIn: USDC_MINT,
  mintOut: SOL_MINT,
  amount: 1000_000_000
});

// Prove & execute atomically
await vault.proveAndExecute(commit);
